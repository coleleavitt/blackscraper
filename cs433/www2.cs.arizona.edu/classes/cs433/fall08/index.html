  • Links have been posted for the GLSL slides and two useful GL Shader Language tutorials.

    You don't need to understand every nuance of GLSL or its syntax, but you should understand a few key concepts about GPUs and GLSL programs that we discussed in class, namely: why we would want to use shaders, and how they fit into/effect the traditional "graphics pipeline"; the basic structure of a GPU program (abstractly speaking), including the two most common types of shaders, the kind of data they each operate on, and the relationship between them; and the communication of data values via qualified variables (in particular, the three qualifiers for C program-to-shader and shader-to-shader communication).

  • Demo 3 has been posted on the demos page. This program illustrates some basic GPU capabilities using GLSL. Includes communication of user-defined variables, as well as hardware texture support.
