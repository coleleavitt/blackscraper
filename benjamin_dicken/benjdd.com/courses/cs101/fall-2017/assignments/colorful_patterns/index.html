Triangle Grid

Again, write a processing program that mimics the below behavior:

A few important things to keep in mind:

  • The canvas is a square (exact size up to you, but must be no shorter than 200 px and no larger than 1000px).
  • Up-to a 16x16 grid of triangles is drawn
  • The size of the grid drawn is determined by the mouse X and Y position.
  • The blue value increases as the mouse moves from the left side to the right side.
  • The red brightness decreases as the mouse moves from the top to the bottom.
  • You must use a nested for-loop to draw the triangles.

HINT: To get the grid-drawing to “follow” the mouse, you can use the mouseX and mouseY variable to control when your for-loops stop.

Name the code file triangle_grid.pde.

BONUS - Resizable Tiles

This program is not required! You may complete if for a small amount of bonus points.

Write a processing program that mimics the below behavior:

A few important things to keep in mind:

  • The canvas is a grid of tiles, 5 tiles wide and 7 tiles tall.
  • The brightness value of the tiles increases from top-to-bottom.
  • The tiles have thick black “grout lines”
  • The 5x7 tile grid scales with the canvas as it is resized!
    • So far, we have not learned how to make a canvas resizable. Everything in the assignments has been a fixed size. To make a canvas resizable, put the following line of code into the setup area: surface.setResizable(true);. The built-in width and height variables update as the canvas is resized, so you can use those to help the tile pattern scale.

Name the code file resizable_tiles.pde.

Comments

Each program file should have a “header” comment at the top that has the following format:

//
// Author: Student Name
// Description:
//    A short description of what this program does!
//

Even though this is a small program, it is important to get in the habit of commenting on your code. This is considered good programming style, and it even more helpful once you start writing larger programs.
